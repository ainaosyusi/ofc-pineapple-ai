## 1. プロジェクト概要

OFC Pineapple（パイナップルオープンフェイスチャイニーズポーカー）3人対戦AIを
深層強化学習（MaskablePPO）で開発するプロジェクト。

### V1 → V2 の経緯

V1（Phase 1〜12, 累計 250M+ ステップ）の全モデルが、C++ evaluator の **ACE=0 バグ**
により**間違ったポーカールールで学習**していたことが判明。ACE（エース）が最弱カードとして
評価されており、AA（エースペア）< 22（2ペア）という致命的な誤り。

さらに外部スペシャリスト 2名のレビューにより、以下の構造的欠陥も発覚：

| # | 課題 | 深刻度 |
|---|------|:------:|
| 1 | **報酬ハッキング**: FL接近ボーナス合計+23 vs フォールペナルティ-6.2 | Critical |
| 2 | **行動空間の意味ブレ**: Street 1 と 2-5 で同じ 243枠に異なる意味 | High |
| 3 | **評価基盤の不在**: 「強さ」を客観評価する仕組みがなかった | High |
| 4 | **Self-Play プール不足**: pool_size=5 では 3人戦のメタが偏る | Medium |
| 5 | **player_0 固定学習**: 座順の癖を過学習するリスク | Medium |

**結論: ゼロからの再構築（V2）が必要**

## 4. Phase 1 Step 1 実験結果

### 4-1. 報酬設定の概要

| 設定 | FL接近報酬 | FL Entry/Stay ボーナス | コンセプト |
|:-----|:----------|:---------------------|:-----------|
| **A. Minimal** | Stage1=0.5, Stage2=1.0, Stage3=0.8 | Entry=30, Stay=60 | V1の1/10に縮小 |
| **B. Zero-shaping** | 全て 0 | Entry=50, Stay=100 | 中間報酬なし |
| **C. Conditional** | A と同値だが非フォール時のみ付与 | Entry=30, Stay=60 | フォール時報酬カット |

### 4-2. 学習曲線（100K → 500K → 1M ステップ）

**フォール率の推移:**

| ステップ | Config A | Config B | Config C |
|:---------|--------:|--------:|--------:|
| 100K | 69.8% | 70.2% | 70.6% |
| 500K | 58.4% | 63.2% | 63.2% |
| **1M** | **49.0%** | **50.8%** | **47.8%** |

**平均スコアの推移:**

| ステップ | Config A | Config B | Config C |
|:---------|--------:|--------:|--------:|
| 100K | -0.64 | +0.00 | -0.54 |
| 500K | +1.56 | +1.43 | +0.14 |
| **1M** | **+2.78** | **+2.66** | **+3.12** |

### 4-3. 1M ステップ時点の最終比較

| 指標 | Config A | Config B | **Config C** | 備考 |
|:-----|--------:|--------:|--------:|:-----|
| **フォール率** | 49.0% | 50.8% | **47.8%** | C が最低 |
| **平均スコア** | +2.78 | +2.66 | **+3.12** | C が最高 |
| **ロイヤリティ** | 0.18 | 0.25 | **0.36** | C が最高 |
| FL Entry | 0.2% | 0.0% | 0.0% | 全Config ほぼ 0%（学習初期） |
| FL Stay | 0.0% | 0.0% | 0.0% | 同上 |
| 勝率 | 32.4% | 32.7% | 32.6% | ほぼ同等 |
| Explained Variance | ~0.11 | ~0.08 | ~0.11 | 全Config 低い（初期段階） |

**報酬ハッキング検出（FL追求時 vs 通常時のフォール率差）:**

| Config | FL追求フォール | 通常フォール | **差分** | 判定 |
|:-------|:-------------|:-----------|:--------|:-----|
| A | 64.9% | 58.8% | **+6.1%** | WARNING (>5%) |
| B | 64.4% | 58.4% | **+6.0%** | WARNING (>5%) |
| C | 64.8% | 58.2% | **+6.6%** | WARNING (>5%) |

→ 全 Config で +6% 前後の WARNING。ただし 1M ステップ時点で FL Entry がほぼ 0% のため、
「FL追求」サンプルの定義が曖昧（ほぼ全ゲームが「FL追求」に分類される）。
**5M+ ステップで FL 行動が明確に分かれてから再評価が必要。**

**ポジション別平均スコア:**

| Config | BTN | SB | BB | 最大差 |
|:-------|----:|---:|---:|------:|
| A | +1.81 | +2.29 | +3.01 | 1.20 |
| B | +3.38 | +2.17 | +3.06 | 1.21 |
| C | +2.42 | +3.27 | +3.06 | 0.85 |

→ 全 Config で座順バイアスは軽微（差 < 2.0）。

### 4-4. Step 1 の暫定結論

**Config C（Conditional shaping）が僅差でリード。**

- フォール率最低（47.8%）
- 平均スコア最高（+3.12）
- ロイヤリティ最高（0.36）
- 「フォールしなかった場合のみ FL段階報酬を付与」が正のインセンティブとして機能

**ただし、1M ステップでの差は小さい。** 3 Config 間のフォール率差は最大 3%、
スコア差は最大 0.46。統計的有意差の確認には 5M+ ステップが必要。

## 5. ベースライン計測（Rule-Based Agent 同士）

150 ゲーム × 各対戦の結果:

| エージェント | vs 対戦相手 | フォール率 | 平均スコア | 勝率 |
|:------------|:----------|--------:|--------:|-----:|
| Safe | Aggressive | 73.3% | +0.8 | 23.3% |
| Safe | Random | 70.0% | -0.7 | 24.0% |
| Aggressive | Safe | 75.3% | -0.3 | 22.0% |
| Aggressive | Random | 72.7% | -0.2 | 23.3% |
| Random | Safe | 65.3% | +0.7 | 27.3% |
| Random | Aggressive | 72.7% | +0.5 | 26.0% |

**特徴:**
- ルールベース同士ではフォール率 65〜75%、スコアほぼ 0 付近
- FL Entry は全エージェントでほぼ 0%
- 1M ステップの学習済みモデルは既にルールベースを上回る（フォール率 48% vs 70%、スコア +3 vs 0）

## 6. 技術的メモ

### 学習環境

| 項目 | 値 |
|:-----|:---|
| 実行環境 | GCP e2-standard-8（8 vCPU, 32GB RAM） |
| ゾーン | asia-northeast2-a（大阪）※ 東京は在庫切れ |
| インスタンス名 | ofc-training-v2 |
| 並列環境数 | NUM_ENVS=4 |
| 学習速度 | ~546 FPS（安定、サーマルスロットリングなし） |
| 1M ステップ所要時間 | 約 31 分 |
| Discord 通知 | 100K ステップごとに進捗通知 |

### 観測空間

891 次元（V1 の 881 次元から +10）:
- カードボード状態（全プレイヤー）
- 手札情報
- FL 拡張特徴量（残りQ/K/A/Joker数、FL outs数）
- current_street One-hot（5次元）

### ネットワーク構成

- MaskablePPO（Stable-Baselines3）
- Policy: [512, 256, 128] × 2（pi/vf 独立）
- lr=3e-4, gamma=0.998, n_steps=8192, batch_size=1024, ent_coef=0.03
